# Calendar UI/UX Audit — 2026-07-05

Scope: `/calendar` page + all flows off it (new booking, view/edit booking, reschedule, block time, day/week/month/agenda views). Lens: premium polish (Linear / Cron / Superhuman bar).

Files audited: `AppShell.tsx`, `calendar/page.tsx`, `DayView.tsx`, `WeekView.tsx`, `MonthView.tsx`, `CalendarSelectorPanel.tsx`, `CalendarChrome.tsx`, `grid.ts`, `new-booking/page.tsx`, `BookingDrawer.tsx`, `globals.css`.

Severity: **P0** broken/exploitable · **P1** hurts premium feel or first impression · **P2** polish. No P0 found — nothing is broken; this is a feel/consistency audit.

Root theme: the three calendar views were built at different times and **don't read as one system**. Month = framed white card with mono headers; Day/Week = edge-to-edge cream with a different type scale. Amber is doing too many jobs. Chrome (sidebar + top bar) has spacing + hierarchy gaps. Fix the system, not 18 one-offs.

## Batch 1 — View surface & cohesion *(your example #2)* — ✅ DONE 2026-07-05

> Shipped: F1 grid→white, F2 Day/Week framed like Month card, F3 now-line→`--now-line` terra token, F4 week arrows equal width. Lint + build pass. **F3 deferred half:** `confirmed` status still shares brand amber — recoloring it is a product-color decision (collides with `completed` green), left for your call.

**F1 · P1 · Day/Week grid sits on cream, not white.** `DayView.tsx:67`, `WeekView.tsx:83` set grid `background: var(--color-cream)`. MonthView day cells are `var(--color-surface)` white (`MonthView.tsx:122`). The week/day time canvas therefore looks flatter and cheaper than month. **Fix:** grid canvas → `var(--color-surface)` (white); keep cream as the page/gutter behind it so the grid reads as a raised sheet. Effort: **S**.

**F2 · P1 · Three views aren't framed the same.** Month wraps its grid in a rounded, shadowed, hairline-bordered card (`MonthView.tsx:84-88`). Day/Week are edge-to-edge with a bare bottom-border header. **Fix:** give Day/Week the same framed-sheet treatment (rounded top corners on desktop, `--shadow-sm`, `--line` border, consistent inset) so switching views feels like the same surface changing, not three apps. Effort: **M**.

**F3 · P1 · Amber overload — no visual hierarchy.** `--color-amber` is simultaneously: primary CTA, active nav, today pill, now-line, selected filter, AND `confirmed` status color (`globals.css:8,33`). When everything is the accent, nothing is. A confirmed booking, the current-time line, and the "New booking" button all shout equally. **Fix:** demote structural chrome (now-line → a quieter terra/neutral or thinner amber; today pill keep amber) and/or give `confirmed` its own hue distinct from the brand accent so status ≠ branding. Effort: **M**.

**F4 · P1 · Week strip is off-center.** Prev arrow column is `width: 44` (`WeekView.tsx:97`), next arrow is `width: 36` (`WeekView.tsx:134`). The 7 day pills are pushed 8px off true center. On a premium grid this is visible. **Fix:** make both nav columns equal width (match the 44px time gutter on both sides). Effort: **S**.

## Batch 2 — Sidebar & filters *(your example #1)*

**F5 · P1 · Sidebar content hugs the left edge.** Search, date, sections all sit at `paddingInline: 12` with the aside itself adding none (`AppShell.tsx:756-764`). Same "too close to the wall" problem you already fought in the nav. Linear/Notion give the rail 16–20px of breathing room. **Fix:** raise sidebar inner gutter to ~16px (and align section-header text with row text). Effort: **S**.

**F6 · P1 · Sidebar loads empty.** VIEW, FILTER, CALENDARS all default **collapsed** (`viewSectionOpen`/`viewMenuOpen`/`calendarsSectionOpen` = false, `AppShell.tsx:318-320`). First impression of the rail is three closed muted rows — dead space. **Fix:** default the **View** section open (it's the highest-frequency control); keep Filter/Calendars collapsed. Effort: **S**.

**F7 · P2 · "Share your booking page" reads as disabled.** Dashed border + muted grey text (`AppShell.tsx:820-822`) — this is your top growth action but it looks like a placeholder. **Fix:** solid subtle surface + amber icon, or a small filled secondary style; keep it clearly tappable. Effort: **S**.

**F8 · P2 · Raw glyph in search clear.** Desktop sidebar search uses a literal `✕` character (`AppShell.tsx:788`) while everything else uses `IconXSmall`. Inconsistent weight/alignment. **Fix:** swap to `IconXSmall`. Effort: **S**.

## Batch 3 — Top bar CTA & notifications *(your example #3)*

**F9 · P1 · Notification bell is bare; badge is cramped.** Bell is a naked 40px hit area (`AppShell.tsx:486-501`); reference has it in a defined bordered circle, more readable. Badge is 16px amber sitting next to the amber CTA — low separation. **Fix:** wrap the bell in a bordered/`--color-cream` container to match the reference; bump badge contrast (e.g. terra/red dot or white ring) so it doesn't blend into the amber CTA beside it. Effort: **S**.

**F10 · P2 · CTA uses a gradient pill.** "New booking" is `--wash-amber` gradient (`AppShell.tsx:477`). The reference "Create" is a flat solid — reads crisper/more modern. **Fix:** try a solid amber (or solid terra) fill; keep the soft shadow. A/B against current. Effort: **S**.

**F11 · P2 · Print button floats unstyled.** Icon-only ghost between the spacer and CTA (`AppShell.tsx:530-538`) with no container — visually orphaned. **Fix:** same bordered-icon container as the bell, or move into an overflow menu. Effort: **S**.

## Batch 4 — Loading, empty & density states

**F12 · P1 · Every range change flashes a full blocking spinner.** `page.tsx:346-350` overlays the whole calendar with `rgba(255,255,255,0.6)` + centered spinner on each week/month step. Initial load correctly uses `CalendarSkeleton`, but navigation feels janky. **Fix:** replace the blocking overlay with a thin top progress bar (or keep the old grid visible + subtle inline spinner). Also the overlay color is hardcoded → token. Effort: **M**.

**F13 · P2 · Week view has no empty state.** DayView shows a "Tap a slot to book" pill when empty (`DayView.tsx:220-226`); WeekView shows nothing — a blank grid. Inconsistent. **Fix:** add a matching quiet hint to WeekView empty weeks. Effort: **S**.

**F14 · P2 · Day view shows the same data twice.** DayView renders the time grid AND a 40%-height agenda list of the same bookings below it (`DayView.tsx:230-244`). Two representations competing = cramped, not premium. **Fix:** pick one as primary (grid) and make the agenda a collapsible/secondary peek, or drop it. Effort: **M**.

## Batch 5 — Polish & tokens

**F15 · P2 · 137 hardcoded hex/rgba outside globals.** Static check flagged 137. Hotspots in scope: time-label color `rgba(30,26,20,0.55)` (`DayView.tsx:119`, `WeekView.tsx:152`), grid lines / block shadows, loading overlay. **Fix:** promote the recurring ones to tokens (`--time-label`, `--grid-overlay`, block shadow → `--shadow-sm`). Effort: **M**.

**F16 · P2 · Booking blocks have no accessible label.** Each block is a `<button>` with only visual text; screen readers get name/time inconsistently and blocked-time buttons are empty. **Fix:** `aria-label` like "Avi Cohen, Haircut, 14:00–14:30, confirmed". Effort: **S**.

**F17 · P2 · No transition between views.** Switching Day↔Week↔Month swaps instantly (`page.tsx:375-420`). A subtle cross-fade would feel considered. **Fix:** light fade/slide on view change. Effort: **S**.

**F18 · P2 · Time gutter typography is thin.** 12px labels, hour-only, `width:44` (`grid.ts:9`, `DayView.tsx:113`). Cron-tier calendars right-align a lighter, smaller label with the line meeting it. **Fix:** refine gutter type scale + alignment once F1/F2 land. Effort: **S**.
